Get up and running with OpenSteer in three different ways: SDK, CUA Agent, or CLI.

<Tabs>
  <Tab title="SDK">
    <Steps>
      <Step title="Install OpenSteer">
        Install the SDK package in your project:
        
        <CodeGroup>
        ```bash npm
        npm install opensteer
        ```
        
        ```bash pnpm
        pnpm add opensteer
        ```
        
        ```bash bun
        bun add opensteer
        ```
        </CodeGroup>
      </Step>
      
      <Step title="Create your first script">
        Create a new file `quickstart.ts` with this complete working example:
        
        ```ts
        import { Opensteer } from "opensteer";

        const opensteer = new Opensteer({ name: "quickstart" });

        try {
          await opensteer.launch();
          await opensteer.goto("https://example.com");

          await opensteer.snapshot({ mode: "action" });
          await opensteer.click({ description: "main call to action" });

          await opensteer.snapshot({ mode: "extraction" });
          const data = await opensteer.extract({
            description: "hero section",
            schema: { title: "string", href: "string" },
          });

          console.log(data);
        } finally {
          await opensteer.close();
        }
        ```
      </Step>
      
      <Step title="Run your script">
        Execute the script:
        
        ```bash
        node quickstart.ts
        ```
        
        The script will:
        1. Launch a browser
        2. Navigate to example.com
        3. Take an action snapshot and click an element
        4. Take an extraction snapshot and extract structured data
        5. Close the browser
      </Step>
      
      <Step title="Understand the key concepts">
        **Namespace (`name`)**: Selectors are persisted in `.opensteer/selectors/quickstart/` for deterministic replay.
        
        **Snapshots**: `mode: "action"` optimizes HTML for interactions, `mode: "extraction"` optimizes for data extraction.
        
        **Descriptions**: Natural language element targeting like `"main call to action"` uses AI to find elements and persists selectors.
        
        **Schema extraction**: Define the structure you want with `{ title: "string", href: "string" }` and OpenSteer extracts it.
      </Step>

  <Tab title="CUA Agent">
    <Steps>
      <Step title="Install OpenSteer">
        ```bash
        npm install opensteer
        ```
      </Step>
      
      <Step title="Set up API key">
        Set your model provider API key:
        
        ```bash
        export OPENAI_API_KEY="your-api-key"
        # or ANTHROPIC_API_KEY, GOOGLE_GENERATIVE_AI_API_KEY
        ```
      </Step>
      
      <Step title="Create CUA agent script">
        Create `agent.ts`:
        
        ```ts
        import { Opensteer } from "opensteer";

        const opensteer = new Opensteer({ 
          model: "openai/computer-use-preview" 
        });

        try {
          await opensteer.launch();
          const agent = opensteer.agent({ mode: "cua" });
          
          const result = await agent.execute({
            instruction: "Go to Hacker News and open the top story.",
            maxSteps: 20,
          });
          
          console.log(result.message);
          console.log("Actions taken:", result.actions.length);
        } finally {
          await opensteer.close();
        }
        ```
      </Step>
      
      <Step title="Run the agent">
        ```bash
        node agent.ts
        ```
        
        The agent will:
        1. Navigate to Hacker News
        2. Identify the top story link
        3. Click on it
        4. Report back with the result
      </Step>
      
      <Step title="Configure agent behavior">
        Customize the agent with options:
        
        ```ts
        const result = await agent.execute({
          instruction: "Find and summarize the top 3 posts",
          maxSteps: 30,           // Allow more steps for complex tasks
          highlightCursor: true    // Visualize agent actions
        });
        ```
      </Step>
    </Steps>
    
    <Warning>
      CUA agents perform autonomous browser actions. Always test with safe websites and appropriate `maxSteps` limits.
    </Warning>
    
    <Info>
      Supported providers: OpenAI (`openai/computer-use-preview`), Anthropic (`anthropic/claude-3-7-sonnet-20250219`), Google (`google/gemini-2.0-flash-exp`)
    </Info>
  </Tab>
  
  <Tab title="CLI">
    <Steps>
      <Step title="Install globally">
        ```bash
        npm i -g opensteer
        opensteer skills install
        ```
      </Step>
      
      <Step title="Open a browser session">
        ```bash
        opensteer open https://example.com --session demo --name quickstart
        ```
        
        This opens a browser, navigates to the URL, and creates a session named `demo` with namespace `quickstart`.
      </Step>
      
      <Step title="Take an action snapshot">
        ```bash
        opensteer snapshot action --session demo
        ```
        
        This returns cleaned HTML with element counters for targeting.
      </Step>
      
      <Step title="Interact with elements">
        ```bash
        opensteer click --description "main call to action" --session demo
        ```
        
        Click elements using natural language descriptions.
      </Step>
      
      <Step title="Extract data">
        First, take an extraction snapshot:
        
        ```bash
        opensteer snapshot extraction --session demo
        ```
        
        Then extract structured data:
        
        ```bash
        opensteer extract '{"title":"string","href":"string"}' --description "hero section" --session demo
        ```
      </Step>
      
      <Step title="Close the session">
        ```bash
        opensteer close --session demo
        ```
      </Step>
    </Steps>
    
    <Tip>
      For CI/CD environments, set `OPENSTEER_SESSION` or `OPENSTEER_CLIENT_ID` environment variables instead of using `--session` flag.
    </Tip>
  </Tab>
</Tabs>

## Best practices for AI agents

If you're building AI agents that generate browser automation scripts:

1. **Use OpenSteer APIs** instead of raw Playwright calls for persistent, replayable scripts
2. **Keep namespace consistent**: SDK `name` must match CLI `--name`
3. **Take snapshots before actions**: `snapshot({ mode: "action" })` before interactions, `snapshot({ mode: "extraction" })` before data extraction
4. **Prefer description targeting**: More maintainable than CSS selectors
5. **Always cleanup**: Wrap in `try/finally` and call `close()`
